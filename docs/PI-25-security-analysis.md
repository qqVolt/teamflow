# PI-25 — Анализ безопасности кода

## Чек-лист безопасности

| # | Проверка | Результат | Комментарий |
|---|---|---|---|
| 1 | HTTPS | N/A | Консольное приложение, веб-сервера нет |
| 2 | Пароли хешированы | КРИТИЧЕСКАЯ УЯЗВИМОСТЬ | Хранились в открытом виде |
| 3 | Защита от SQL-инъекций | OK | InMemoryStorage, при миграции нужны параметризованные запросы |
| 4 | Проверка авторизации | УЯЗВИМОСТЬ | Отсутствует middleware |
| 5 | Валидация ввода | Частично | validate() есть, но XSS-защита только при веб-интерфейсе |
| 6 | Singleton корректность | OK | DatabaseConnection и InMemoryStorage |

## Найденные уязвимости

### Уязвимость 1. Пароли в открытом виде (КРИТИЧЕСКАЯ)
**Файл:** `src/models/User.js`, поле `#password`
**Проблема:** `this.#password = data.password || ''` — пароль хранился как есть.
**Решение:** Внедрено bcrypt-хеширование.

### Уязвимость 2. Отсутствие авторизации (ВЫСОКАЯ)
**Файл:** `src/services/TaskService.js`
**Проблема:** Нет проверки, кто выполняет действие.
**Решение:** Создан `src/middleware/AuthMiddleware.js`.

### Уязвимость 3. Нет защиты от подбора (СРЕДНЯЯ)
**Файл:** `src/models/User.js`
**Проблема:** `checkPassword()` не ограничивает попытки.
**Решение:** Рекомендуется rate limiting и блокировка.

### Уязвимость 4. API-ключ в коде (ДЛЯ ПРОДАКШЕНА)
**Файл:** `src/patterns/GoogleCalendarAdapter.js`
**Решение:** Хранить в переменных окружения.
